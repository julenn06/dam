# Guía de Contribución

¡Gracias por tu interés en contribuir a este proyecto educativo! Este documento proporciona las pautas para contribuir al repositorio.

## 🤝 Cómo Contribuir

1. **Fork** el repositorio
2. Crea una **rama nueva** (`git checkout -b feature/nuevaCaracteristica`)
3. Realiza tus **cambios**
4. **Commit** tus cambios (`git commit -am 'Añade nueva característica'`)
5. **Push** a la rama (`git push origin feature/nuevaCaracteristica`)
6. Crea un nuevo **Pull Request**

## 📝 Convenciones de Código

### Java
- Usa camelCase para nombres de variables y métodos
- PascalCase para nombres de clases
- Documentación JavaDoc en clases y métodos públicos

### JavaScript/TypeScript
- Usa const/let (evita var)
- Nombres descriptivos en camelCase
- Documenta funciones con JSDoc

### C#
- PascalCase para nombres de clases y métodos públicos
- camelCase para variables privadas
- Documentación XML en miembros públicos

## 🔍 Proceso de Review

1. Asegúrate de que tu código compila sin errores
2. Incluye tests si es necesario
3. Actualiza la documentación relevante
4. Espera la revisión de los mantenedores

## 📚 Estructura de Carpetas

Mantén la estructura existente del proyecto:
```
DAM1/
  └── [Asignatura]/
      └── [Evaluación]/
          └── [Proyecto]
DAM2/
  └── [Asignatura]/
      └── [Evaluación]/
          └── [Proyecto]
```

## ⚠️ Guías Importantes

1. **Commits**
   - Mensajes claros y descriptivos
   - Un commit por cambio lógico
   - Referencia issues si aplica

2. **Documentación**
   - Actualiza README si es necesario
   - Documenta cambios importantes
   - Incluye ejemplos si aplica

3. **Testing**
   - Añade tests para nuevo código
   - Verifica tests existentes
   - Documenta casos de prueba

## 🐛 Reportando Issues

- Usa las plantillas proporcionadas
- Incluye pasos para reproducir
- Adjunta logs/screenshots relevantes
- Etiqueta apropiadamente

## 📮 Contacto

Para dudas o sugerencias:
- Abre un issue
- Contacta con los mantenedores
